★Vimの学習#

◇序文#

  1. 移行

    ヒント ターミナルで vimtutor を実行して、Vimの基本コマンドを学びましょう。この対話式チュートリアルは約30分で、基礎知識をカバーしています。

    Vimの文法構造を理解することが、その強力な機能をマスターするための鍵です。少数のコマンドを学び、直感的に組み合わせることで、様々な操作を効率的に完了できます。

    編集速度が元の50%に達したら、完全にVimに切り替える時です。

    Vimの学習は自転車の乗り方を学ぶようなものです。ガイドを読むだけでは不十分で、実際に練習して初めてマスターできます。

    Vimをマスターする鍵は実践にあります：コマンドを繰り返し入力して組み合わせ、その機能を探索し、:helpと検索エンジンを上手く活用しましょう。目標はコマンドの実行を自然で本能的なものにすることです。

    Vimヘルプマニュアルを使うための小技：特定のモードでのコマンドを検索する場合、モードのプレフィックスを付ける必要があります。例えば、挿入モードでのCtrl-Pを検索するには、:h i_CTRL-Pと検索します。ここでi_は挿入モードを表します。

  2. Vimrc

    • 行番号の設定：

      • 行番号を永続的に表示するには、Vim設定ファイル（.vimrc）にset numberを追加します。
      • まだ.vimrcファイルがない場合は、ルートディレクトリに作成できます。通常は.vimrcという名前です。macOSでは、パスは~/.vimrcです。
      • ファイルを保存（:w）した後、:source %を使用して設定を有効にすると、行番号が各行の左側に表示されます。

    • 一時的な設定：

      • 永続的に設定したくない場合は、Vimコマンドラインで:set numberと入力できますが、これは一時的な効果しかなく、Vimを閉じると設定は消えます。

    • Vim固有の機能を有効にする：

      • Vimの固有機能を使用するには、.vimrcにset nocompatibleを追加する必要があります。そうしないと、多くのVim機能が無効になります。

    • 一般的な設定方法：

      • Vimの任意の設定オプションについては、.vimrcファイルに追加し、保存して有効にするだけです。

◇はじめの一歩#

  1. モードを理解する：Vimには異なるモードがあります：

    • ノーマルモード：ナビゲーションとコマンド用（Vimを開いたときのデフォルトモード）
    • 挿入モード：テキスト入力用（iを押して入る）
    • ビジュアルモード：テキスト選択用（vを押して入る）
    • コマンドモード：コマンド実行用（:を押して入る）

  2. 保存と終了

    • :w：保存（:write）
    • :w file.txt：新しいファイルとして保存
    • :q：終了（:quit）
    • :wq：保存して終了
    • :q!：保存せずに終了

  3. ヘルプ

    • :helpまたは:h：メインヘルプファイルを開く
    • :help コマンド：特定のコマンドのヘルプを見る（例：:h write-quit）

  4. ファイルを開く

    • vim ファイル名：指定したファイルを開く
    • vim file1 file2：複数のファイルを開く（バッファに配置）

  5. パラメータ

    • vim --version：Vimのバージョン情報とサポートされている全機能を表示、+はサポートされている機能、-はサポートされていない機能
    • :version：Vimプログラム内で現在のVimバージョンを確認
    • hello.txtファイルを開いた直後にコマンドを実行する方法はいくつかあります：

      • :sコマンド（substitueの省略形）を使用してテキストを置換

      • +コマンド構文を使用：

        vim +コマンド hello.txt

        例えば、hello.txtを開いた直後に、すべての"pancake"を"bagel"に置換

        vim +%s/pancake/bagel/g hello.txt
        • vimはVimエディタを起動
        • %sは全文置換、gはグローバル（各行のすべての一致を置換）

        このコマンドは重ねることができ、ファイル内のすべての元の"pancake"、“bagel”、“egg"が"donut"に変わります：

        vim +%s/pancake/bagel/g +%s/bagel/egg/g +%s/egg/donut/g hello.txt

        テキストを検索：

        vim +/検索語 hello.txt

      • -cパラメータを使用：

        vim -c 'コマンド' hello.txt

        vim -c %s/pancake/bagel/g hello.txt
vim -c %s/pancake/bagel/g -c %s/bagel/egg/g -c %s/egg/donut/g hello.txt

  6. 複数のウィンドウを開く

    水平分割ウィンドウ（-oオプション）、垂直分割ウィンドウ（-Oオプション）も同様。

    • Vimを開き、2つの水平分割された空のウィンドウを作成（上下に配置）。 
      vim -o2
    • Vimを開き、5つの水平分割された空のウィンドウを作成。 
      vim -o5
    • Vimを開き、5つの水平分割されたウィンドウを作成。最初の2つのウィンドウにはそれぞれhello1.txtとhello2.txtが読み込まれ、残りの3つのウィンドウは空のウィンドウ。 
      vim -o5 hello1.txt hello2.txt

    これらのコマンドは、複数のファイルや同じファイルの異なる部分を同時に表示・編集する必要がある場合に特に役立ちます。Vimのウィンドウナビゲーションコマンド（Ctrl+wと方向キーなど）と組み合わせることで、異なるウィンドウ間を効率的に切り替えることができます。

  7. サスペンド

    • Vimでエディタをサスペンドする方法は3つあります：

      1. Ctrl-zショートカットを押す
      2. :stopコマンドを使用
      3. :suspendコマンドを使用

    • サスペンドされたVimセッションに戻るには、ターミナルで以下を入力するだけです：

      fg

      fgコマンドは"foreground"の略で、最近サスペンドされたジョブをフォアグラウンドに復帰させます。

    • 複数のサスペンドされたジョブがある場合は、jobsコマンドですべてのサスペンドされたジョブのリストを表示し、fg %nコマンドで特定のジョブを復帰させることができます。ここでnはジョブ番号です。例：

      fg %2

    • 使用シーン

      • ファイルの内容を見る必要があるがVimを終了したくない場合
      • 情報を得るためにいくつかのコマンドを実行し、その結果をVimに持ち帰る場合
      • 現在の編集状態を失うことなく緊急タスクを処理する必要がある場合

  8. Vim起動テクニック

    • コマンドラインパラメータ

      • Vimは他のターミナルコマンドと同様に、様々な起動オプションとフラグを受け付けます
      • よく使うオプション：+{cmd}/-c cmdで起動時にVimコマンドを実行できます
        • vim -c "set number" file.txt（ファイルを開いて行番号を表示）
        • vim +10 file.txt（10行目にジャンプ）

    • パイプ操作

      • Vimはパイプとリダイレクトと組み合わせて使用できます
      • ls -l | vim -：コマンド出力をVimにリダイレクトして編集
      • grep pattern file | vim -：検索結果を直接編集

    • リソース

      • man vim：ターミナルドキュメント
      • :help：内部ヘルプシステム
