★Learning Vim#

◇Preface#

  1. Transition

    Tip Run vimtutor in the terminal to learn basic Vim commands. This interactive tutorial takes about 30 minutes and covers the fundamentals.

    Understanding Vim’s syntactic structure is key to mastering its powerful features. You only need to learn a small set of commands and combine them intuitively to efficiently complete various operations.

    Once your editing speed reaches 50% of your original speed, it’s time to fully switch to Vim.

    Learning Vim is like learning to ride a bicycle - reading guides alone isn’t enough; you must practice hands-on to master it.

    The key to mastering Vim is practice: repeatedly input and combine commands, explore its features, make good use of :help and search engines. The goal is to make command execution natural and instinctive.

    A small tip for using the Vim help manual: when looking for commands in specific modes, add the mode prefix. For example, to search for Ctrl-P in insert mode, search for :h i_CTRL-P, where i_ represents insert mode.

  2. Vimrc

    • Setting line numbers:

      • To permanently display line numbers, add set number to your Vim configuration file (.vimrc).
      • If you don’t have a .vimrc file yet, create one in your root directory. On macOS, the path is ~/.vimrc.
      • After saving the file with :w, use :source % to activate the configuration. Line numbers will appear on the left side of each line.

    • Temporary settings:

      • If you don’t want to set it permanently, you can type :set number in Vim’s command line, but this is only temporary and will disappear after closing Vim.

    • Enable Vim-specific features:

      • To use Vim-specific features, you must add set nocompatible to your .vimrc, otherwise many Vim features will be disabled.

    • General configuration method:

      • For any Vim configuration option, simply add it to your .vimrc file, save, and activate it.

◇Getting Started#

  1. Understanding modes: Vim has different modes:

    • Normal mode: for navigation and commands (default mode when opening Vim)
    • Insert mode: for entering text (press i to enter)
    • Visual mode: for selecting text (press v to enter)
    • Command mode: for executing commands (press : to enter)

  2. Saving and exiting

    • :w: save (:write)
    • :w file.txt: save as a new file
    • :q: quit (:quit)
    • :wq: save and quit
    • :q!: quit without saving

  3. Help

    • :help or :h: open the main help file
    • :help command: view help for a specific command (e.g., :h write-quit)

  4. Opening files

    • vim filename: open a specified file
    • vim file1 file2: open multiple files (place them in buffers)

  5. Parameters

    • vim --version: display Vim version information and all supported features, + indicates supported features, - indicates unsupported features
    • :version: check the current Vim version within the Vim program
    • There are several ways to execute commands immediately after opening the hello.txt file:

      • Use the :s command (short for substitute) to replace text

      • Use the +command syntax:

        vim +command hello.txt

        For example, to replace all “pancake” with “bagel” immediately after opening hello.txt:

        vim +%s/pancake/bagel/g hello.txt
        • vim starts the Vim editor
        • %s means full text replacement, g means global (replace all matches in each line)

        Commands can be stacked; all original “pancake”, “bagel”, and “egg” in the file will become “donut”:

        vim +%s/pancake/bagel/g +%s/bagel/egg/g +%s/egg/donut/g hello.txt

        Searching for text:

        vim +/searchword hello.txt

      • Using the -c parameter:

        vim -c 'command' hello.txt

        vim -c %s/pancake/bagel/g hello.txt
vim -c %s/pancake/bagel/g -c %s/bagel/egg/g -c %s/egg/donut/g hello.txt

  6. Opening multiple windows

    Horizontal split windows (using the -o option), vertical split windows (using the -O option) work similarly.

    • Open Vim and create 2 horizontally split empty windows (arranged top and bottom). 
      vim -o2
    • Open Vim and create 5 horizontally split empty windows. 
      vim -o5
    • Open Vim and create 5 horizontally split windows, where the first two windows load hello1.txt and hello2.txt respectively, and the remaining 3 windows are empty. 
      vim -o5 hello1.txt hello2.txt

    These commands are particularly useful when you need to view and edit multiple files or different parts of the same file simultaneously. Combined with Vim’s window navigation commands (such as Ctrl+w plus direction keys), you can efficiently switch between different windows.

  7. Suspending

    • There are three ways to suspend Vim:

      1. Press the Ctrl-z shortcut
      2. Use the :stop command
      3. Use the :suspend command

    • To return to a suspended Vim session, simply type in the terminal:

      fg

      The fg command is short for “foreground” and will restore the most recently suspended job to the foreground.

    • If you have multiple suspended jobs, you can use the jobs command to view a list of all suspended jobs, then use the fg %n command to restore a specific job, where n is the job number. For example:

      fg %2

    • Use cases

      • Need to view file content without exiting Vim
      • Need to run some commands to get information, then bring the results back to Vim
      • Need to handle urgent tasks without losing the current editing state

  8. Vim startup tips

    • Command line parameters

      • Vim accepts various startup options and flags, similar to other terminal commands
      • Common options: +{cmd}/-c cmd can execute Vim commands at startup
        • vim -c "set number" file.txt (open file and display line numbers)
        • vim +10 file.txt (jump to line 10)

    • Pipe operations

      • Vim can be combined with pipes and redirections
      • ls -l | vim -: Redirect command output to Vim for editing
      • grep pattern file | vim -: Edit search results directly

    • Resources

      • man vim: Terminal documentation
      • :help: Internal help system
